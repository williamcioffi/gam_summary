some notes

filtering gaps

We’ve just messing around with this and haven’t thought about filtering too much on the input data, but there are some comments:

  • 70 1803 has a big gap during the exposure
  • 79 1807 has a big gap during the exposure
  • 70 1802 this is right at the beginning of the record and because we concat dives that are less than 800 meters into surf this turns into a surf. But surfs at the beginning of a record are onbounded on the left side, so this should be deleted. this is an oversight in the prep functions that can be corrected. also this would be the single data point for that exposure so probably would filter this out anyway.

Because the way we’ve set this up it doesn’t matter if there are gaps as long as there is some data during the +24 hour period, these don’t fail or anything, but probably we want to filter out tag*exp events that are mostly gap (we can think of some threshold) especially if that gap is in the first half of the window where it matters the most.
